# 拆解对象：《鸣潮》长离 全技能组

### 一、基础信息

| 维度     | 内容                                                         |
| :------- | :----------------------------------------------------------- |
| 角色定位 | 速切副C                                                      |
| 核心机制 | 普攻最后一段特殊普攻，技能后释放特殊普攻叠层，空中释放特殊普攻，以及大招直接获得四层，四层满后长按普攻释放强化重击 |
| 特殊机制 | **离火**：施放普攻**心眼·征**命中目标时，获得1层【离火】。<br/>施放普攻**心眼·冲**命中目标时，获得1层【离火】。<br/>施放共鸣解放**离火照丹心**时，获得4层【离火】。<br/>**离火层数视觉反馈**：每层离火都会让长离的建模发生一定程度的视觉变化，四层时视觉效果达到峰值。反推为每层离火GE附带一个独立的`GameplayCue`，或根据离火层数切换材质实例参数。 |

### 二、普攻连段

| 段数            | 表现/观察                              | 伤害类型                                   | 反推实现                                                     |
| :-------------- | :------------------------------------- | :----------------------------------------- | :----------------------------------------------------------- |
| 第1段           | 挥剑斜砍                               | 扇形范围                                   | GA_Changli_Attack_01：`Play Montage And Wait` 播放普攻1动画，`Wait Gameplay Event` 等待 `AnimNotify` 触发伤害帧，`ApplyGameplayEffectToTarget` 结算扇形重叠检测伤害 |
| 第2段           | 幻化凤羽斜砍                           | 圆形范围                                   | GA_Changli_Attack_02：继承上段连段索引，圆形Overlap检测，命中施加伤害GE |
| 第3段           | 本体带凤羽向前旋转攻击                 | 圆形范围带牵引                             | GA_Changli_Attack_03：圆形碰撞+牵引效果（`Add Impulse` 向玩家方向拉怪），`GameplayCue` 播放凤羽特效 |
| 第4段           | 收回凤羽且往后退举伞，剑竖置于身侧     | 圆形范围                                   | GA_Changli_Attack_04：圆形Overlap，动作末尾的 `AnimNotify` 触发蓝图事件，给自身角色添加临时Tag `State.Changli.ComboStep5`，作为特殊普攻的输入窗口 |
| 第5段(特殊普攻) | 既可以通过普攻释放，也可以通过跳跃释放 | 地面：向前突刺横砍。空中：化为凤凰向上冲。 | GA_Changli_Attack_05：`Activation Required Tags` 必须包含 `State.Changli.ComboStep5` 才能激活（该Tag由第4段末尾赋予，或由共鸣技能释放后赋予的 `State.Changli.SkillLanded` 提供）。地面/空中根据角色状态走不同动画分支；命中为自己施加1层【离火】GE |

**反推总结**：

- 连段之间穿插闪避会从头开始，而完美闪避会从第3段起——反推为普攻GA的 `Retrigger Delay` 设置：完美闪避时GA被中断但保留当前连段索引；普通闪避则触发 `End Ability`，下次激活GA时连段计数器归零。
- 特殊普攻并非独立按键技能，而是受限于上一段普攻赋予的临时Tag窗口。这种“连段→窗口Tag→下一段检查Tag”的模式，若窗口内未按下普攻，标签过期自动回收，连段重置。
- 特殊普攻有两个触发来源：普攻第4段末尾的窗口Tag，或共鸣技能释放后赋予的窗口Tag（`State.Changli.SkillLanded`）。二者提供相同的激活条件，实现“普攻连段→特殊普攻”和“技能→特殊普攻”的双入口设计。空中版特殊普攻同理，依赖空中普攻第4段末尾的 `State.Changli.AirComboStep5` 窗口Tag接续。

### 三、空中普攻

触发条件：角色处于空中状态（`Character Movement` 的 `Is Falling` 为 True），按下普攻键。

| 段数  | 表现/观察              | 伤害类型 | 反推实现                                                     |
| :---- | :--------------------- | :------- | :----------------------------------------------------------- |
| 第1段 | 横剑                   | 矩形范围 | GA_Changli_AirAttack_01：`Play Montage And Wait` 播放空中普攻1，矩形碰撞检测，命中施加伤害GE |
| 第2段 | 丢出剑旋转后收回       | 圆形范围 | GA_Changli_AirAttack_02：剑飞行阶段生成独立投射物Actor（圆形碰撞+自动旋转动画），飞回时销毁 |
| 第3段 | 向斜上方丢剑旋转后收回 | 矩形范围 | GA_Changli_AirAttack_03：与第2段同逻辑，投射物方向改为斜上方 |
| 第4段 | 收回剑直立于身前       | 矩形范围 | GA_Changli_AirAttack_04：收剑动画+矩形伤害判定，动作末尾赋予自身 `State.Changli.AirComboStep5` 标签，用于接续空中版特殊普攻；同时赋予 `State.Changli.AirComboFinished` 标签（短暂窗口），作为下落攻击的激活条件 |

**反推总结**：
- 空中可以闪避且无论是否完美都能继续衔接普攻——说明空中普攻GA的 `Activation Blocked Tags` 不包含闪避Tag，且闪避不会触发 `End Ability`，连段索引保留在黑板中。
- 空中普攻和地面普攻共用同一套普攻GA但 `Montage` 资产不同——蓝图内根据 `Is Falling` 分支选择动画。

### 四、重击/下落攻击/空中特殊普攻

#### 4.1 重击

| 触发条件 | 表现/观察                                        | 反推实现                                                     |
| :------- | :----------------------------------------------- | :----------------------------------------------------------- |
| 重击     | 斜劈、横劈、上挑，后若继续释放普攻则进入空中普攻 | GA_Changli_HeavyAttack：`CommitAbilityCost` 检查体力（Cost GE扣体力，Duration = Instant），`Play Montage And Wait` 播放重击三段动画。第三段给自身一个向上的 `Add Impulse`，后续普攻自动进入空中连段 |

#### 4.2 下落攻击

| 触发条件                 | 表现/观察 | 反推实现                                                     |
| :----------------------- | :-------- | :----------------------------------------------------------- |
| 空中普攻完成后，长按普攻 | 举剑下劈  | GA_Changli_PlungeAttack：`Activation Required Tags` 包含 `State.Changli.AirComboFinished`（空中普攻特殊攻击末尾赋予，窗口约1.0–1.5秒）。落地帧 `Overlap` 范围检测+`Camera Shake` |

#### 4.3 空中特殊普攻

| 触发条件                                                 | 表现/观察        | 反推实现                                                     |
| :------------------------------------------------------- | :--------------- | :----------------------------------------------------------- |
| 空中普攻第4段后，在窗口内按下普攻；或在有窗口Tag时起跳后 | 化为凤凰向前冲刺 | 与地面特殊普攻共用 `GA_Changli_Attack_05`，`Activation Required Tags` 包含 `State.Changli.AirComboStep5` 或 `State.Changli.SkillLanded`。蓝图内检测角色处于空中状态，走空中动画分支；`GameplayCue` 播放凤凰变身特效；命中为自己施加1层【离火】GE |

**反推要点**：
- 重击消耗体力——反推为Cost GE（扣体力，Duration = Instant）。
- 重击第三段完成后给自身赋予的 `State.Changli.AirLaunch` 标签，是空中普攻GA的 `Activation Required Tags` 之一，实现“重击→空中普攻”的无缝衔接。
- 空中释放技能/特殊普攻时的**浮空保持**：当长离在空中释放共鸣技能或空中特殊普攻时，角色并非在播放动画的过程中任由重力拉坠，而是通过 `AnimNotify` 将 `Character Movement` 的 `Movement Mode` 暂时切换为 `Flying`（飞行模式）并将速度归零，技能/特殊普攻动画结束后在 `End Ability` 中恢复为 `Falling`，从而获得“定在半空出招”的视觉表现。

### 五、共鸣技能

| 维度         | 分析                                                         |
| :----------- | :----------------------------------------------------------- |
| **观察**     | 幻成虚影在前方固定范围内斩击                                 |
| **冷却时间** | 12s                                                          |
| **伤害范围** | 前方矩形空间                                                 |
| **特殊机制** | 开局拥有两层充能，12s恢复一层，释放完后可直接接入特殊普攻    |
| **反推实现** | GA_Changli_Skill：`CommitAbilityCooldown`（CD GE，12s）+ `CommitAbilityCost`（消耗一层充能）。充能系统反推为独立的 `FGameplayAttributeData` 属性“共鸣技能充能层数”，最大值2，通过 `Wait Delay` 每12秒 `ApplyGameplayEffectToSelf` 回复一层（GE Duration = Instant）。技能释放后，`AnimNotify` 赋予自身 `State.Changli.SkillLanded` 标签，作为后续特殊普攻窗口的激活条件。开局自带两层充能——在角色 `BeginPlay` 时 `SetNumericAttributeBase` 直接初始化充能层数为2。 |

### 六、共鸣解放

| 维度         | 分析                                                         |
| :----------- | :----------------------------------------------------------- |
| **观察**     | 镜头聚焦到长离面部，然后长离向下攻击                         |
| **激活条件** | 能量条满才能放（反推为独立FGameplayAttributeData属性“共鸣能量”，通过普攻/技能命中GE回复，大招Cost GE消耗） |
| **伤害机制** | 单段高伤                                                     |
| **反推实现** | 能量满→玩家按键调用激活函数→GA_Changli_Ultimate：<br/>1. `CommitAbility` 检查能量GE消耗+CD<br/>2. `Play Montage And Wait` 播放大招动画（包含特写镜头→定序器片段控制镜头聚焦面部→向下攻击）<br/>3. 动画过程中的 `AnimNotify_GameplayCue` 触发大招专属特效<br/>4. 伤害判定帧：`ApplyGameplayEffectToTarget` 结算单段高伤<br/>5. 命中后 `ApplyGameplayEffectToSelf` 为自己施加4层【离火】GE<br/>6. `Activation Owned Tags` 加上 `State.Invincible`，大招期间免疫伤害<br/>7. `End Ability` |

### 七、核心被动

| 触发条件                   | 效果                             | 反推实现                                                     |
| :------------------------- | :------------------------------- | :----------------------------------------------------------- |
| 拥有四层[离火]时，长按普攻 | 提高20%热熔伤害且忽略敌方15%防御 | GA_Changli_EnhancedHeavy：激活条件不能写在 `Activation Required Tags` 里依靠 `State.Changli.LihuoStack4` 这种标签来判断——GE堆叠本身不会动态切换Tag。正确的做法是重写该GA的 `CanActivateAbility()` 函数，在C++中获取自身ASC，调用 `GetGameplayEffectCount(GE_Changli_LihuoStack)` 查询当前离火GE的层数是否 $\ge 4$，满足则返回true。<br/>激活后：<br/>1. `Play Montage And Wait` 播放强化重击专属动画<br/>2. 伤害帧 `ApplyGameplayEffectToTarget`——高倍率伤害GE，其 `Execution Calculation` 中直接读取离火层数作为倍率系数，并内置 `Multiply` 热熔伤害1.2倍、`Override` 忽略防御15%的属性处理。Buff效果与伤害结算合并在同一个Execution内部完成，不依赖独立的Buff GE手动添加和移除，杜绝因网络延迟导致伤害结算时Buff已被提前删除的风险。<br/>3. `End Ability`——技能结束时，由系统底层自动清理该GA实例持有的所有瞬时资源。离火GE层数在强化重击结算后通过Execution内部的消耗逻辑清零。 |

### 八、整体总结

**这个角色的技能组设计逻辑**：
长离作为速切副C定位，核心玩法是“攒离火→清空离火打爆发”。她的离火获取来源有三个——特殊普攻命中（1层/次）、共鸣技能接特殊普攻（1层/次）、大招（一次性4层）。但若要一次打满4层离火再退场会浪费大量站场时间，因此我认为最佳循环为“登场接第3段→第4段→特殊普攻攒1层→接技能→特殊普攻再攒1层→切回主C；下次登场时视充能情况选择开大（直接叠4层打强化重击）或继续用技能+特殊普攻循环叠层”。整体属于速切爆发型角色，不站场、不吃资源，核心伤害集中在强化重击上，其余时间由主C填充。
