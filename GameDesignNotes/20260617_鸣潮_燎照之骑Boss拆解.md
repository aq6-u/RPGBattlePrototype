# 拆解对象：《鸣潮》燎照之骑

### 一、基础信息

| 维度      | 内容                                                         |
| :-------- | :----------------------------------------------------------- |
| Boss 定位 | 使用武器打近战，攻击风格偏激进。一阶段骑摩托高频压制，二阶段下摩托后转为本体近战。 |
| 核心机制  | 双韧性条。第一条韧性条为摩托韧性条，击破后Boss丢弃摩托进入二阶段；第二条为本体韧性条。二阶段部分技能出现反击圈，提示玩家可弹反。 |
| 特殊资源  | 两条独立的韧性条。反推为两个独立的 `FGameplayAttributeData` 属性——`MotorbikeStagger`（摩托韧性）和 `BodyStagger`（本体韧性）。普攻/技能命中时Apply对应的扣减GE。`MotorbikeStagger` 归零触发转阶段GA，`BodyStagger` 归零触发瘫痪GA。 |

### 二、行为状态与阶段切换

| 阶段               | 触发条件         | 行为模式变化                                                 | 反推实现                                                     |
| :----------------- | :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| 第一阶段（骑摩托） | 默认             | 场地中等候玩家进入后触发。骑摩托移动，速度较快。使用一阶段技能组攻击。 | 行为树主分支：追击→根据距离和玩家位置选择一阶段技能。移动速度从一阶段DataTable读取。 |
| 第二阶段（下摩托） | 摩托韧性条被击破 | 丢弃摩托，技能组完全不同。移速下降，但攻击范围和伤害提升。部分技能出现反击圈提示。 | 行为树用Decorator检测 `MotorbikeStagger <= 0`。转阶段时强制中断当前行为，播放摩托销毁动画→`SpawnActor` 生成投掷摩托→切换到二阶段行为树分支。移速和攻击参数从二阶段DataTable读取。 |
| 特殊状态           | 无               | 无                                                           | —                                                            |

**反推总结**：

- 阶段切换检测：不是血量阈值，是**摩托韧性条归零**。反推为行为树的 Decorator 持续监听 `MotorbikeStagger` 属性值。
- 进入新阶段时Boss强制中断当前行为——反推为转阶段GA激活时，对一阶段的所有GA执行Cancel（通过 `Cancel Abilities with Tag` 批量终止）。

### 五、受击与韧性反馈

| 受击类型     | Boss 反应                      | 反推实现                                                     |
| :----------- | :----------------------------- | :----------------------------------------------------------- |
| 普通攻击命中 | 无硬直                         | 一阶段全程霸体——Boss身上常驻 `State.Unstoppable` Tag。所有受击GE在 `PostGameplayEffectExecute` 里检测到此Tag时不触发硬直逻辑，只扣减韧性条。 |
| 连续攻击命中 | 无硬直                         | 同上。                                                       |
| 震协击破     | Boss进入硬直停顿               | 震协条满时按F触发震协攻击能触发Boss短暂停顿，反推为震协条满时F键事件触发强制硬直GA。 |
| 弹反成功     | Boss进入硬直停顿               | 玩家振刀成功→给Boss Apply一个带 `State.Parried` Tag的GE→行为树检测到此Tag时暂停攻击分支→播放硬直动画。硬直时长从DataTable读取。 |
| 转阶段       | Boss强制中断当前行为，丢出摩托 | 摩托韧性条归零→触发转阶段GA→对一阶段所有GA执行Cancel→播放转阶段动画→生成投掷摩托→切换到二阶段行为树。 |

**反推总结**：

- Boss全程无常规受击硬直——说明一阶段和二阶段都有霸体保护。反推为常驻 `State.Unstoppable` Tag 挂在Boss身上。
- 只有弹反成功和韧性条相关机制才能造成停顿——硬直的触发条件不是伤害量，而是特定的GameplayEvent（弹反成功事件、韧性条归零事件）。
- 霸体期间：所有受击GE在 `PostGameplayEffectExecute` 里检测到 `State.Unstoppable` Tag时，不触发硬直动画，只扣减韧性条。

### 六、整体总结

**这个 Boss 的设计逻辑**：

燎照之骑给玩家的核心体验是“分阶段的近战压制”。一阶段骑摩托高频攻击，玩家需要在密集的蓄力劈击中找到输出窗口。二阶段下摩托后攻击变慢但威胁更高，同时出现反击圈引导玩家主动弹反——从一阶段的“闪避为主”切换到二阶段的“弹反为主”。双韧性条的设计让转阶段有明确的视觉和机制节点，而不是隐式的血量阈值。

**如果我需要在 UE5 里复现这个 Boss**：

- **GA（至少 11 个）**：一阶段5个技能GA + 二阶段5个技能GA + 转阶段GA（含摩托投掷）
- **GE（至少 15 个）**：各技能伤害GE × 10、击飞GE、摩托韧性扣减GE、本体韧性扣减GE、弹反成功反馈GE、转阶段触发GE
- **GameplayTag（至少 6 个）**：`State.Unstoppable`（全程霸体）、`State.Parried`（被弹刀硬直）、`State.Phase2`（二阶段标记）、`Ability.Boss.MotorbikeSkill`（一阶段技能分类Tag，供转阶段Cancel使用）、`Ability.Boss.BodySkill`（二阶段技能分类Tag）、`Event.StaggerBreak`（韧性条击破事件）
- **行为树分支（至少 4 个）**：一阶段攻击序列（包含身后检测触发不同技能）、二阶段攻击序列、转阶段中断分支、弹反硬直暂停分支
- **DataTable配置项**：十个技能的伤害数值、十个技能的前摇时间、击飞距离和持续时间、摩托韧性条最大值和每次扣减值、本体韧性条最大值和每次扣减值、弹反窗口时长、弹反硬直时长、一阶段移速、二阶段移速、各技能的碰撞体矩形尺寸
