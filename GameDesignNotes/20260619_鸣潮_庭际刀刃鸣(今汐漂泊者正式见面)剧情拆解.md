# 拆解对象：《鸣潮》主线—庭际刀刃鸣(今汐漂泊者正式见面)

## 1. 镜头切换分析

#### 一、开场 / 对话阶段

整篇对话都处于月光下，二人身体外处都有一层白光描边，能给玩家眼睛一亮的感觉，也能体现出二人作为正派的神圣感。  
推测实现：通过后处理材质（Post Process）或角色材质中的自发光边缘光（Rim Light）叠加而成，在定序器中可控制强度曲线，使角色在夜色中仍保持视觉焦点。

**反推实现**：
整段对话的镜头调度核心是**定序器（Level Sequence）的 Camera Cuts Track**。每个景别对应的是一台提前摆放好的 Camera Actor，通过轨道上的关键帧决定其切入切出时间。
- **固定机位**（近景 / 中景）直接使用静态 Camera Actor，在 Camera Cuts Track 中按顺序排列即可。
- **运动镜头**（如逐渐靠近今汐、环绕二人等）则在对应 Camera Actor 上为 Transform 添加关键帧动画，或在定序器内启用 **Look At 轨道** 让镜头自动跟随角色头部骨骼。
- **月光描边**效果可通过两种方式实现：在角色材质中叠加一层菲涅尔边缘光（Rim Light），并在定序器中暴露参数曲线，根据情绪需要微调强度；或通过后处理材质针对角色所在的自定义深度通道（Custom Depth）绘制边缘光，确保只在角色身上生效。
- **城镇空镜**（展现煌珑）很可能是一台独立的高角度 Camera Actor，其切入时不需要角色在画面中，纯粹起情绪烘托作用。

整段过场没有进入/退出战斗的逻辑，因此定序器内不需要事件轨道触发战斗解锁；但可在结尾处添加一个 Event Track 调用“恢复玩家控制”的蓝图节点（通常为 Set Input Enabled）。

#### 二、情绪推进 / 场景转换阶段

从“今汐转身、走向栏杆”开始，镜头开始向外延伸，逐渐引入今州城镇的概貌，将情绪从二人的私密交谈拉升到“守护一方”的宏大叙事。

| 景别        | 画面内容                   | 功能分析                                                     |
| :---------- | :------------------------- | :----------------------------------------------------------- |
| 近景        | 今汐转身，走向栏杆         | 动作本身是一个“离开对话圈、靠近观察点”的信号，引导玩家视线跟随，为接下来的场景展示做准备 |
| 中景        | 今汐手扶栏杆，背景显现城镇 | 角色由交谈者变为“守夜人”形象，中景同时保留角色身体语言和城镇背景，让担忧情绪既有载体又有依据 |
| 远景 / 空镜 | 展示煌珑城镇全景（月光下） | 纯粹的“环境叙事”：用宁静的画面反衬今汐口中的危机，让玩家意识到“这份安宁可能不复存在”，情绪从好奇转向沉重 |

**反推实现**：
- **转身动作**：在定序器中为角色添加 **Animation Track**，播放预先制作的“走向栏杆”动画片段（约 2-3 秒），该动画可以是通用移动动画加一个收尾手扶栏杆的叠加层。
- **城镇空镜**：为一台独立的高角度 Camera Actor 设置较长的焦距，并在其切入前短暂重叠（Blend）前一个近景镜头，形成“角色看向哪里，镜头就切向哪里”的心理引导。
- **情绪曲线控制**：这一阶段的镜头持续时长应比前半段略长（每个镜头 4-6 秒），让玩家有时间消化场景信息。

#### 三、叙事引导 / 关键物体聚焦阶段

全段有两处明显的“关键物体聚焦”：今汐背部的声痕，以及多次出现的今州城镇。前者是世界观的核心符号，后者是角色的情感锚点。

| 景别/镜头类型        | 画面内容                                             | 功能分析                                                     |
| :------------------- | :--------------------------------------------------- | :----------------------------------------------------------- |
| 中至近景（运动镜头） | 聚焦今汐背面，展示声痕，背景为今州山河，镜头逐渐靠近 | 明确告知玩家“声痕”是重要设定，同时镜头推近模拟了漂泊者（以及玩家）的好奇心 |
| 近景                 | 今汐占右下角，镜头大面积留给脚下的城镇               | 身份代表责任，城镇代表情感；此构图将两者并置，突出今汐的美好品质 |
| 中景                 | 镜头不动，固定                                       | 回答“今汐在守护什么”这个问题，完成情感铺垫                   |

**反推实现**：
- **推近声痕**：定序器中给该 Camera Actor 的 Transform 添加位置关键帧（从稍远推至近处）。
- **人物让出画面**（今汐占右下角）：实际上并未切换摄像机，而是通过调整同一 Camera Actor 的构图角度或角色站位实现。

#### 四、收尾 / 情绪回落阶段

最后五个镜头（从“今汐担忧的神情”到“今汐稍微有点担心的面部表情”）构成闭环，将情绪从“接纳任务”的高点逐步拉回内敛，留下余韵。

| 景别/镜头类型 | 画面内容               | 功能分析                                                     |
| :------------ | :--------------------- | :----------------------------------------------------------- |
| 近景          | 今汐担忧的神情         | 卸下令尹的威严，露出私人情感，让玩家看到“坚强下的脆弱”       |
| 近景          | 漂泊者认可的神情       | 一个肯定的眼神，完成情感交换，同时给予玩家“被需要的满足感”   |
| 近景          | 今汐稍微有点担心的表情 | 担忧中带着一丝暖意，情绪不再沉重，而是偏温柔的悬念——为后续剧情埋下期待 |

**反推实现**：
- **表情微调**：这三个近景的表情变化幅度很小，不可能用完整动画片段实现。更可能的做法是——在定序器中对角色面部添加 **Float Track（浮点曲线轨道）**，分别控制眉梢角度、嘴角弧度和眼神亮度，由策划在时间轴上直接拽曲线调整，无需动画师介入。
- **镜头切换节奏**：收尾阶段三个镜头之间用硬切，营造“情绪结晶”的节奏感。
- **结束恢复控制**：定序器末尾的 Event Track 触发蓝图事件，调用 `Set Input Enabled`（或等价的玩家控制器函数），让玩家恢复操作。此时可以顺便触发一条系统提示或任务更新UI。

## 2. 动作与表情分析

**整体观察**：  
整段过场的角色运动可分为两类：  
1. **大幅度身体动作**（回头、迈步、走向栏杆、转身）——由独立的短动画片段驱动，在定序器（Level Sequence）中通过 **Animation Track** 触发。  
2. **细微表情与眼神变化**（失望、坚定、认可、担忧、眼神交流）——几乎不依赖完整动画，而是通过 **Morph Target 曲线**（眉毛、嘴角、眼神方向等）在定序器内由策划直接调参。  
口型与语音的同步，推测由 **Audio Track** 驱动 **Morph Target 嘴型参数** 实时解算，或使用预烘焙的嘴型动画序列叠加。全程没有需要玩家输入的战斗逻辑，故所有动作表现均由定序器集中控制，无需在角色蓝图写死硬编码。

**反推实现**：  
- **大幅度动作**（回头、迈步、转身、走向栏杆）：这些是独立预录的动画片段（动画蒙太奇或短序列），在定序器中通过 **Animation Track** 挂在对应角色上。行走类动画通常拆分为“起→走→停”三段，利用 Blend 节点平滑过渡，策划只需在时间轴上拖动片段的入点出点。  
- **细微表情**（眼神坚定、失望、认可）：完全不依赖完整动画，而是通过定序器的 **Float Track（浮点曲线轨道）**直接控制角色面部 Morph Target 参数。例如，“失望”时降低嘴角曲线、抬高眉心曲线；“坚定”时拉高眼神亮度、收紧嘴唇。策划可在时间轴上肉眼观察表情变化并即时调整。  
- **眼部特写与视线**：眼部有独立的 Morph Target 参数（瞳孔大小、眼睑开合），同时视线方向可通过 **Look At 轨道** 设置目标点（漂泊者头部骨骼），让角色眼球随对方移动而自然跟踪。  
- **口型同步**：若该片段有语音，反推为 **Audio Track** 驱动 Morph Target 嘴型参数实时解算，或使用事先烘焙的嘴型动画序列，作为叠加层与身体动画混合。

**反推实现**：  
- **走向栏杆**：行走动画与对话阶段相同处理。手扶栏杆的动作可能是一个单独的半身动画叠加在待机姿势上（上半身播放扶栏杆动画，下半身保持静止），通过动画分层或蒙太奇实现，避免为每个过场制作全身动画。  
- **推近镜头**：给对应 Camera Actor 的 Transform 添加位置关键帧（向前平移）。
- **展望城镇**：角色不需大幅动作，但可在定序器中为今汐添加 **Look At 轨道**，将其视线目标设置为远处城镇的某个虚拟点，确保目光方向与镜头构图一致。

## 3. 音乐与音效分析

| 时间点/片段                  | 音乐/音效内容                                                | 功能分析                                                     |
| :--------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| 整段对话                     | 平稳、悠扬的今州城镇夜间主题音乐，以吹奏乐器和弦乐为主，贯穿全程，无明显段落切换或情绪转折 | 奠定静谧、庄重的氛围；统一各镜头下的情绪，避免因频繁切换镜头而产生听觉割裂。全程不换曲、不推高情绪，说明这段过场不希望音乐“抢戏”——情绪完全由台词、表情和镜头语言承担 |
| 今汐转身走向栏杆，手扶栏杆时 | 隐约的脚步声                                                 | 强化动作真实感，填补台词空白时刻，让玩家感知角色物理存在     |

**反推实现**：
- **全程单曲策略**：在定序器中通过 **Audio Track** 放置一段 2D 的非空间化音频资源（如 .wav 或 Sound Cue），设置为一次性播放，长度覆盖整个过场。首尾配合淡入淡出关键帧即可，中间无需任何切换或参数变动。
- **不做动态混音的原因推测**：这段过场以对话为主，且全程处于同一场景，没有场景切换或情绪剧烈翻转，因此不需要通过音乐变化来暗示信息。保持音乐稳定，反而让玩家更专注于今汐的台词和表情细节。
- **脚步声**：由动画自带的 **Anim Notify** 触发，定序器无需额外配置。
- **语音处理**：对话语音通过高优先级音频单独播放（**Dialogue Voice** 或独立 **Audio Track**），引擎的 Sound Mix 可能预设了语音触发时自动压低背景音乐的基础音量（Ducking），但这是全局规则，并非本段过场专有。

## 4. 情绪曲线与设计意图总结

**整体情绪曲线**：  
静谧开场（月光下的尊敬与礼貌）→ 疑惑与紧张（今汐透露危机、转身迈步）→ 沉重与担忧（展示声痕、城镇全景，凸显责任）→ 温暖与释然（漂泊者认可，二人约定秘密，眼神中略带笑意）。情绪从平和进入，逐步攀升到严肃紧张，再通过环境叙事缓和，最终落回温柔余韵，形成“起—承—转—合”的完整呼吸。

**设计意图**：  
这段过场是今汐角色塑造的基石——通过双人对话与场景空镜的结合，在短短几分钟内完成“介绍危机背景、介绍漂泊者、确立今汐令尹形象、建立玩家与今汐的情感连接”四重任务，为后续主线埋下情感伏笔。

**可借鉴的设计技巧**：

1. **用环境叙事代替台词**：在对话中插入今州城镇空镜，让玩家从“听人讲”变成“亲自看到”，增强情感共鸣。该手法可在项目的重要对话中使用，用场景画面呼应角色话语。
2. **用镜头推近引导视线**：展现声痕时，镜头缓缓推近，模拟人类“凝视”心理。后续若做关键道具或线索展示，可直接复用这种“推镜”的镜头语言。
3. **用表情代替台词完成情绪转换**：今汐从失望到坚定的转变、漂泊者从无奈到认可的转变，均通过近景表情连续切接实现，没有一句旁白。这证明了“正反打表情对切”可以高效代替复杂的内心独白，适合在对话向过场中使用。
4. **全程保持一种音乐烘托，降低混乱感**：仅用一首主旋律贯穿始终，避免了频繁切歌带来的情绪错乱。对于时长较短的单幕过场，这是低成本且有效的声音策略。
5. **Morph Target 驱动的微表情复用法**：今汐在收尾阶段的“担忧中带暖意”表情，可以通过调节“嘴角上扬”和“眉间紧锁”两组参数的数值比例实现，无需制作全新动画。今后在自己的项目过场中，可以为重要角色设定几个“情绪基态参数组”，在定序器中快速调用。
