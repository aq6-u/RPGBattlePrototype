# **拆解对象**：《鸣潮》第一章剧情—北落野大决战

## **1. 镜头切换分析**

#### 一、剧情对话阶段：

| 景别 | 画面内容                 | 功能分析                                                     |
| :--- | :----------------------- | :----------------------------------------------------------- |
| 中景 | 漂泊者与忌炎面对面交谈   | 交代两人站位和周围环境，让玩家建立“在哪”的空间认知           |
| 近景 | 漂泊者的动作（表示决心） | 通过收紧景别，聚焦于角色的肢体语言和微表情，传递“决心”的情绪信号 |
| 近景 | 忌炎的动作（表示决心）   | 与上一个镜头形成对称结构，两个近景连续出现，暗示两人形成了共鸣或呼应 |
| 中景 | 二人背面，面前是决战入口 | 景别重新拉开，让玩家从“看人”切换到“看场景”，自然地将注意力引向下一个目标——决战入口 |

**反推实现**：

- 这段在定序器（Level Sequence）中用 4 台固定 Camera Actor 在 **Camera Cuts Track（镜头切换轨道）** 上按顺序切换即可。中景→近景→近景→中景的序列形成一个完整的“呼吸节奏”，不需要运动镜头。

------

#### 二、进入决战场地：

| 景别 | 画面内容              | 功能分析                                                     |
| :--- | :-------------------- | :----------------------------------------------------------- |
| 远景 | 决战场地全貌          | 建立新场景的空间感，告诉玩家“战斗地点”，同时以宏大感制造心理压迫 |
| 中景 | 二人的震惊表情        | 从宏大的环境切回角色反应，用中景同时保留角色肢体和部分环境，让震惊感既有情绪来源又有环境依据 |
| 近景 | 鸣式核心              | 景别收紧到具体物体，告诉玩家“这个东西是场景中最重要的”       |
| 近景 | 第一个 Boss，二人位于 | 近景拍 Boss + 角色渺小身影在后方，经典的“渺小人类面对庞然大物”构图，在此阶段完成情绪从震惊到压迫的升级 |
| 近景 | 二人面前              | 近景拍二人动作，体现决战时的紧张氛围，也是表现为即将到来的战斗做准备，给玩家即将开战的信号 |
| 近景 | Boss 回头             | 前一个镜头已经把 Boss 拍得足够大，此时 Boss 回头的动作成为视觉冲击点 |
| 远景 | 瞬间拉远              | 从近景跳切到远景，强烈的反差制造了“战斗开始”的宣告感。这个节点，正好是系统解锁玩家输入的时机 |

**反推实现**：

- 这段的 6 个镜头沿着一条清晰的情绪曲线编排——环境建立（远景）→ 情绪承接（中景）→ 焦点收束（近景核心）→ 压迫递增（近景 Boss + 角色在后方）→ **挑衅瞬间（Boss 回头近景）**→ **战斗发令（拉远）**。在定序器中的实现依然是机位硬切。从技术策划角度看，**“Boss回头”和“瞬间拉远”这两个镜头的衔接**，不仅是视觉上的爆发，也是代码层面的关键节点——可能需要在这个时间点触发解锁玩家输入的事件。

------

#### 三、战斗阶段转换：

- 阶段切换时的 CG 播片，技术上承担了“**加载掩体**”的功能。在 CG 播放期间，系统可以安全地切换 Boss 的行为树阶段、重新配置 GAS 技能，而不被玩家感知到任何卡顿或跳变。

------

#### 四、剧情收尾阶段：

| 景别/镜头类型   | 画面内容                           | 功能分析                                                     |
| :-------------- | :--------------------------------- | :----------------------------------------------------------- |
| 中景            | 二人背面，仰望鸣式                 | 让玩家重新看到这个环境的全貌和角色的站位关系                 |
| 近景            | 二人对话                           | 收束剧情线，用近景让玩家聚焦于角色的最终对话                 |
| 特写/跟随镜头   | 镜头给到阿布，跟随阿布飞行进入声痕 | 这是整个过场中最关键的叙事引导镜头。镜头从“拍人”切换到“拍线索”，告诉玩家“这个东西很重要”。而且这里不是硬切，是摄像机在做跟随运动——反推为定序器中给摄像机设置了跟随阿布的位移动画，或者直接将摄像机 Attach 到阿布的骨骼上 |
| 中景 + 镜头抖动 | 二人背面，镜头抖动表示崩塌         | 景别回到中景，但加入了镜头抖动曲线。这个抖动不是特效，而是在定序器里手动给摄像机添加了 Transform 关键帧或 zitter 曲线，让玩家在视觉上感知到“崩塌正在发生” |
| 近景            | 二人对话                           | 崩塌结束后回到近景，重新聚焦于角色，为 CG 结尾做情绪铺垫     |
| CG              | 结束                               | 收尾播片                                                     |

**反推实现**：

- 这段的技术含量最高，因为它同时包含了**硬切**（近景对话部分）和**运动镜头**（跟随阿布部分）。镜头抖动则是在定序器中给摄像机添加了额外的位移曲线。三套不同的镜头操作（硬切、跟随路径、抖动曲线）在同一个定序器时间轴上同时运作，需要镜头轨道、角色轨道、特效轨道在时间上严格对齐。

## **2. 动作与表情分析**

**整体观察**：这段过场中，凡是有台词的段落，角色嘴部都在实时运动，与语音同步。猜测这是通过 **Morph Target 嘴型混合**或者直接播放对口型动画实现的。核心**控制权都在定序器的动画轨道和曲线轨道上**，而不是在角色蓝图中写死逻辑。

------

#### 一、剧情对话阶段：

| 景别 | 画面内容                 | 动作与表情（观察）                           | 功能分析                                                     |
| :--- | :----------------------- | :------------------------------------------- | :----------------------------------------------------------- |
| 中景 | 漂泊者与忌炎面对面交谈   | 忌炎先看向漂泊者，漂泊者随后回看             | 眼神交流建立人物之间的关系——忌炎作为“将军”先投来注视，是一种确认 |
| 近景 | 漂泊者的动作（表示决心） | 漂泊者做出表示决心的肢体动作（如握拳或抬头） | 情绪重音。景别收紧的同时，动作幅度放大，两者叠加将角色的内心状态外化给玩家 |
| 近景 | 忌炎的动作（表示决心）   | 忌炎点头，表情坚定                           | 正反打对位。漂泊者表达决心后，忌炎的点头是“认可”信号，完成了一次完整的情感交流闭环 |
| 中景 | 二人背面，面前是决战入口 | 无明显动作                                   | 此时无动作比有动作更重要——让玩家静止地凝视入口，制造“暴风雨前的宁静” |

**反推实现**：

- 对话阶段的三个近景动作（漂泊者决心、忌炎点头、坚定表情），如果是独立的、不和位移耦合的动作，反推为**叠加在基础待机姿势上的短动画蒙太奇**。这种“上半身播动画+下半身停住”的模式在定序器里通过动画轨道分层即可实现，不需要给每个角色做整套过场动画。
- 眼神和表情（尤其是嘴型）反推为 Morph Target 驱动——在定序器中给角色面部添加 **Float Track（浮点曲线轨道）**，分别控制嘴型开合、眉毛角度、眼神方向的参数。策划可以在时间轴上直接调整这些曲线，不需要动画师重新K帧。忌炎的表情全程变化较少，可能只用了 2-3 组 Morph Target 参数，调起来很快。

#### 二、进入决战场地：

| 景别 | 画面内容                        | 动作与表情（观察）       | 功能分析                                                     |
| :--- | :------------------------------ | :----------------------- | :----------------------------------------------------------- |
| 远景 | 决战场地全貌                    | 无                       | 环境叙事，不做人物动作                                       |
| 中景 | 二人的震惊表情                  | 二人向前走几步，然后停住 | “走→停”这个动作是情绪泄压的起点——从走动到突然静止，暗示“前方有不可置信的景象”。在情绪曲线上，这是“震惊”的肢体前置信号 |
| 近景 | 鸣式核心                        | 无                       | 焦点转移到物体，人物退场                                     |
| 近景 | 第一个 Boss，二人位于 Boss 身后 | 无                       | 压迫感构图阶段，不需要动作                                   |
| 近景 | 二人面前                        | 二人做出防御姿态         | 这是整个战斗前情绪的最高点。“防御姿态”是身体告诉玩家“危险就在眼前”的通用语言。与之前的“震惊停步”形成情绪递进：震惊→防御→准备战斗 |
| 近景 | Boss 回头                       | 无                       | 挑衅瞬间由 Boss 发起，角色此时无需动作                       |
| 远景 | 瞬间拉远                        | 无                       | 视觉发令枪                                                   |

**反推实现**：

- “二人向前走几步然后停住”这个动作，反推为一个简短的位移动画片段。在定序器中，每个角色的位移动作是独立轨道控制的——走和停的时间点可以分别调整，确保两人同时停下。
- “防御姿态”反推为一个通用受击预备动画。这种动画在多个Boss战过场中可以复用——只需要换场景和Boss，角色的“防御姿态”动作是一样的。这是过场演出的**模块化思维**：把高频出现的动作抽成独立片段，策划在定序器里直接调用，不需要每次做新动画。

#### 三、剧情收尾阶段：

| 景别/镜头类型 | 画面内容                   | 动作与表情（观察）                     | 功能分析                                                     |
| :------------ | :------------------------- | :------------------------------------- | :----------------------------------------------------------- |
| 中景          | 二人背面，仰望鸣式         | 无                                     | 战斗结束后的场景再确认，让玩家消化刚才的经历                 |
| 近景          | 二人对话                   | 二人表情坚定                           | “坚定”这个表情是收尾的情绪基调——告诉玩家“故事还没结束”，为后续版本埋下情绪伏笔 |
| 特写/跟随镜头 | 阿布飞入漂泊者声痕         | 二人表情惊讶，视线跟随阿布移动         | 这里的表情变化是关键叙事信号——“惊讶”说明二人对阿布的行为并不完全理解。视线跟随的同步动作，让玩家的注意力从“角色在看什么”自然转向“我也要看看那个东西” |
| 中景+镜头抖动 | 二人背面，镜头抖动表示崩塌 | 无                                     | 环境反馈阶段，动作退场，让镜头抖动传递体感                   |
| 近景          | 二人对话                   | 忌炎看向漂泊者说话，漂泊者点头表示肯定 | 完成了一次从“达成共识”的角色关系,表示二人关系进一步提升      |
| CG            | 结束                       | 无                                     | 收尾播片                                                     |

**反推实现**：

- “二人表情惊讶+视线跟随阿布”是这一段实现难度最高的部分。表情方面，反推为 Morph Target 驱动——惊讶需要同时调整眉毛抬起、嘴型微张、眼睛瞪大三组参数。视线跟随方面，反推为角色眼神方向的实时追踪——在定序器中给角色添加 **Look At 轨道**，让角色的眼球跟随阿布的飞行轨迹实时转动。
- “忌炎看向漂泊者说话，漂泊者点头”这段，是整个过场在“动作层面”的最终收束。和前文对话阶段的处理方式一致——Morph Target 控制眼神和表情，叠加轻微的头部位移动画。点头动作幅度很小，通常可以直接用 Morph Target 的颈部参数微调，不一定需要专门的动画片段。

## **3. 战斗**

#### 第三阶段boss战：

##### 1. Boss飞天激光柱（追踪型地面AOE）

| 维度             | 分析                                                         |
| :--------------- | :----------------------------------------------------------- |
| **观察**         | Boss飞天，朝地面发射激光柱，追踪玩家位置。地面上没有看到贴花预警，感觉像是射穿到地面下 |
| **表现形式分析** | 这里没有常规的“红圈预警”，可能是出于两种考虑：1. 美术风格统一，不想让贴花破坏地面纹理；2. 激光速度适中，不需要预警。 |
| **功能分析**     | 塑造最终阶段的压迫感，强制玩家在战斗最后阶段保持高频移动     |
| **反推实现**     | 激光柱本质是一个**持续移动的碰撞体**。流程推测：Boss飞天→播放蒙太奇→ `SpawnActor` 生成激光柱 Actor。激光柱每帧 `Tick` 跟随玩家位置移动。伤害端：激光柱Actor自带的球体/胶囊碰撞体设为 `Overlap`，每次 `Overlap` 事件触发时**检查去重数组（防同一帧多次伤害）**，扣血通过 `ApplyGameplayEffectToTarget` 实现。若激光有灼烧地面痕迹，则是命中时在对应位置动态生成一个Decal Actor |

------

##### 2. 弹幕反弹 + 强制切人（演出与机制结合）

| 维度             | 分析                                                         |
| :--------------- | :----------------------------------------------------------- |
| **观察**         | 强制切换为忌炎（开大后状态），Boss发射弹幕，玩家反弹弹幕造成伤害。每次打击有顿帧（Hitstop）和屏幕抖动反馈，但感觉不像无限触发，可能设置了N秒内最多触发M次抖动 |
| **表现形式分析** | 强制切人：在过场中直接切换玩家操控角色，这是用 Level Sequence 事件轨道触发角色切换逻辑。弹幕反弹：弹幕的反弹判定、反弹后的伤害归属、弹幕的飞行轨迹变化，涉及两套碰撞检测（玩家反弹框 + 弹幕与Boss的碰撞）。顿帧和屏幕抖动：有频率限制，说明有一个“**连续命中反馈抑制**”机制，防止高频触发导致视觉疲劳或性能问题 |
| **功能分析**     | 给予玩家“用Boss的武器打Boss”的爽感，强制切人让玩家体验忌炎开大后的完整状态，提升对角色和剧情的投入度 |
| **反推实现**     | 强制切人：定序器里的事件轨道 → 触发角色切换蓝图（通过解除 `Possess` 并由新角色执行 `Possess`）→ 切换到忌炎，并立刻调用 `Give Ability And Activate Once` 给他开大后的GA和GE。弹幕反弹流程：弹幕飞向玩家→玩家身上有一个不可见的反弹碰撞体（和武器胶囊体类似的 `Overlap` 判定）→击中反弹碰撞体后，弹幕的移动方向取反或重新计算反射向量→弹幕改为对Boss造成伤害。**顿帧/屏幕抖动限频**：在触发顿帧的蓝图里加一个 Bool 变量（如 `bCanTriggerShake`）作为触发开关，每次触发后设为 `False`，通过 `Set Timer by Event` 延迟0.2秒后再设回 `True`——这就实现了“N秒内最多M次”的限制。这样做确保了无论弹幕命中多快，都不会造成视觉上的疲劳 |
